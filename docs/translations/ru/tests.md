# Тесты

В проекте используется `pytest`.

## Запуск полного набора

```bash
pytest
```

## Запуск отдельной части

```bash
pytest tests/test_service.py
pytest tests/test_cli.py
```

Тесты покрывают:

- парсинг CLI и коды выхода;
- оркестрацию и обработку ошибок;
- абстракцию провайдера;
- helper-функции Markdown-рендеринга;
- поведение HTTP-клиента.

## Примечание для контрибьюторов

Репозиторий устроен так, чтобы публичный API можно было тестировать без реальной сети. Собственные провайдеры, сессии и workflow можно подменять напрямую в тестах.

Описание публичного API и точек расширения находится в [docs/translations/ru/api.md](api.md).
